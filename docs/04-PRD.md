# Twinturing 产品需求文档 (PRD)

**版本**: v1.0  
**日期**: 2026-03-29  
**品牌**: Twinturing（双生图灵）  
**产品**: 人造金刚石饰品独立站

---

## 一、产品概述

### 1.1 产品定位

Twinturing是一个面向海外客户的人造金刚石饰品DTC独立站，提供高品质、环保、价格透明的实验室培育钻石珠宝。

### 1.2 目标用户

| 用户类型 | 描述 | 核心需求 |
|----------|------|----------|
| **准新人** | 25-35岁，准备订婚/结婚 | 高性价比钻戒、仪式感 |
| **时尚女性** | 25-40岁，注重个人风格 | 日常珠宝、独特设计 |
| **礼物购买者** | 30-50岁，男性为主 | 送礼选择、品质保证 |
| **海外华人** | 25-45岁，中文偏好 | 中文服务、信任感 |

### 1.3 核心价值主张

1. **Same Sparkle, Smarter Choice** — 同样的闪耀，更智慧的选择
2. **价格透明** — 比天然钻石省70-80%
3. **环保可追溯** — 100%实验室培育，零冲突
4. **双语原生** — 中英文同等体验

## 二、MVP版本（Phase 1）

### 2.1 MVP目标

**目标**: 3周内上线静态展示网站，验证市场需求  
**部署**: Vercel（静态生成）  
**范围**: 纯展示，无电商闭环

### 2.2 MVP功能清单

| 功能模块 | 具体功能 | 优先级 |
|----------|----------|--------|
| **首页** | 品牌故事、核心产品展示、CTA | P0 |
| **产品展示** | 产品列表、分类筛选、产品详情 | P0 |
| **关于我们** | 品牌故事、团队、价值观 | P0 |
| **联系我们** | 联系表单、社交媒体链接 | P0 |
| **博客/资讯** | 钻石知识、选购指南 | P1 |
| **FAQ** | 常见问题解答 | P1 |
| **多语言** | 中英文切换 | P0 |
| **SEO优化** | Meta标签、结构化数据、sitemap | P0 |
| **响应式** | 移动端适配 | P0 |

### 2.3 MVP产品展示

**初始SKU**: 20-30个产品
| 品类 | 数量 | 价格区间 |
|------|------|----------|
| 订婚钻戒 | 10-12款 | $500 - $3,000 |
| 项链 | 6-8款 | $300 - $1,500 |
| 耳环 | 4-6款 | $200 - $800 |

### 2.4 MVP技术选型

| 层级 | 技术 | 原因 |
|------|------|------|
| **框架** | Next.js 14 (App Router) | SSR/SSG支持，SEO友好 |
| **样式** | Tailwind CSS | 快速开发，响应式 |
| **部署** | Vercel | 免费额度，自动CI/CD |
| **图片** | Cloudinary | 自动优化，CDN加速 |
| **内容** | MDX/JSON | 无需数据库，版本控制 |
| **表单** | Formspree/Resend | 无需后端 |
| **分析** | Vercel Analytics + Google Analytics | 免费 |

### 2.5 MVP页面结构

```
/ (首页)
├── /products (产品列表)
│   ├── /products/rings (戒指)
│   ├── /products/necklaces (项链)
│   └── /products/earrings (耳环)
├── /products/[slug] (产品详情)
├── /about (关于我们)
├── /contact (联系我们)
├── /blog (博客)
│   └── /blog/[slug] (文章详情)
├── /faq (常见问题)
├── /en (英文版)
└── /zh (中文版)
```

## 三、生产环境版本（Phase 2）

### 3.1 Phase 2目标

**目标**: MVP验证后，构建完整电商闭环  
**时间**: MVP上线后3-6个月  
**范围**: 完整电商功能

### 3.2 Phase 2功能清单

| 功能模块 | 具体功能 | 优先级 |
|----------|----------|--------|
| **用户系统** | 注册/登录、个人中心、订单历史 | P0 |
| **购物车** | 添加/删除、数量修改、价格计算 | P0 |
| **支付系统** | Stripe集成、多币种、分期付款 | P0 |
| **订单管理** | 订单创建、状态跟踪、发票生成 | P0 |
| **库存管理** | SKU管理、库存同步、低库存预警 | P0 |
| **物流集成** | 运费计算、物流跟踪、国际配送 | P0 |
| **客服系统** | 在线客服、工单系统 | P1 |
| **营销工具** | 优惠券、满减、会员折扣 | P1 |
| **评价系统** | 用户评价、图片上传 | P1 |
| **推荐系统** | 相关推荐、个性化推荐 | P2 |

### 3.3 Phase 2技术升级

| 层级 | 技术 | 说明 |
|------|------|------|
| **后端** | Node.js + Express/Fastify | API服务 |
| **数据库** | PostgreSQL + Prisma | 数据持久化 |
| **缓存** | Redis | 会话、库存、热点数据 |
| **队列** | Bull/BullMQ | 异步任务 |
| **部署** | Vercel + AWS RDS | 前端+后端分离 |
| **监控** | Sentry + Datadog | 错误追踪、性能监控 |

### 3.4 Phase 2数据模型

```sql
-- 核心实体
User (用户)
Product (产品)
Variant (产品变体：尺寸、材质)
Order (订单)
OrderItem (订单项)
Payment (支付记录)
Shipment (物流记录)

-- 营销
Coupon (优惠券)
Review (评价)
Wishlist (收藏)

-- 内容
BlogPost (博客)
Page (静态页面)
FAQ (常见问题)
```

## 五、非功能性需求

### 5.1 性能

| 指标 | MVP目标 | 生产目标 |
|------|---------|----------|
| 首屏加载 | <3s | <1.5s |
| Lighthouse分数 | >80 | >90 |
| 可用性 | 99% | 99.9% |

### 5.2 安全

- HTTPS强制
- CSP (Content Security Policy)
- 支付PCI DSS合规
- 用户数据GDPR合规
- 定期安全审计

### 5.3 SEO

- 结构化数据（Product, Article, FAQ）
- 自动生成sitemap.xml
- Open Graph + Twitter Card
- 多语言hreflang标签
- 语义化HTML

### 5.4 可访问性

- WCAG 2.1 AA级别
- 键盘导航
- 屏幕阅读器支持
- 色彩对比度达标
